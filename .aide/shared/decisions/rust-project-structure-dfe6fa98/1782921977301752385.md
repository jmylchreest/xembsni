# rust-project-structure

**Decision:** Use Cargo workspaces with crates/ directory; separate lib.rs from main.rs even in binaries

## Rationale

Workspaces provide dependency and lint inheritance. Separating lib.rs from main.rs enables unit testing without running the binary and allows library reuse.

## Details

Layout: Cargo.toml (workspace root), crates/core/ (types/traits), crates/cli/, crates/server/. Simple projects: lib.rs + thin main.rs (parse args, wire deps, call lib). Use workspace = true for deps, metadata, lints. Set publish = false for internal crates. Commit Cargo.lock for binaries only.
