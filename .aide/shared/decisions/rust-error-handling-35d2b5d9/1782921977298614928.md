# rust-error-handling

**Decision:** Use thiserror for library error types, anyhow for application error propagation; never use unwrap() in production code paths

## Rationale

thiserror provides zero-cost derive macros for std::error::Error without appearing in your public API. anyhow provides easy context addition with .context(). unwrap() panics on failure — use ? to propagate or expect() with a reason when panic is justified.

## Details

Libs: #[derive(thiserror::Error)] enums, #[from] for conversion, #[source] for chains. Apps: anyhow::Result + .context(). Alt: snafu (rich selectors), miette (CLI diagnostics). Enable clippy::unwrap_used in prod, allow in tests. Panic only via expect("reason").
