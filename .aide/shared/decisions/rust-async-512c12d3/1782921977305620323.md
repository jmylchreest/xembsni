# rust-async

**Decision:** Use tokio as the async runtime; keep domain logic synchronous; async only at I/O boundaries

## Rationale

Tokio is the dominant async runtime (20,000+ crates depend on it). async-std is discontinued (March 2025). Async adds complexity — only use it when you need high-concurrency I/O. Keeping domain logic synchronous makes it testable and composable.

## Details

#[tokio::main] for apps. tokio::spawn_blocking for CPU work. .instrument(span) for async tracing context. Prefer multi-threaded runtime; current_thread for constrained envs. Anti-patterns: blocking in async (use spawn_blocking), unnecessary async, missing Send + 'static on spawned tasks.
