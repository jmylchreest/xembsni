# rust-logging

**Decision:** Use tracing for all logging and observability; never use println! or the log crate directly

## Rationale

tracing is the de facto standard for Rust logging, developed by the Tokio team. It supports structured logging, spans, async context propagation, and OpenTelemetry integration. It supersedes the log crate.

## Details

tracing-subscriber for config (fmt layer, JSON, EnvFilter for RUST_LOG). tracing-log bridge for log-crate deps. Structured fields: info!(user_id = %id, "msg") -- no string interpolation. .instrument(span) on futures. tracing-opentelemetry for distributed tracing.
