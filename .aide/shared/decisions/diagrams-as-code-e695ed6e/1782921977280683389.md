# diagrams-as-code

**Decision:** Store diagrams as text-based DSL source under version control, reviewed in PRs like code

## Rationale

Text-format diagrams are diffable, searchable, and evolve with the code they describe. Binary exports from a drag-and-drop drawing tool drift within weeks and cannot be reviewed for correctness.

## Details

Pick a text DSL that fits the diagram type and team familiarity — Mermaid, PlantUML, D2, and Structurizr are common choices. Rendering is a project-level choice this decision does not prescribe: commit an SVG next to the source, render on-demand locally, wire up a CI renderer, or render nothing at all are all acceptable. The one rule: the source lives in the repo next to what it documents.
