# rust-performance

**Decision:** Profile before optimizing; use criterion or divan for benchmarks; track regressions in CI

## Rationale

Micro-benchmarks can be misleading — always profile realistic workloads first. criterion provides statistical analysis with historical tracking. divan is the modern alternative with simpler API and less timing noise in CI.

## Details

criterion: HTML reports. divan: CI-friendly (less noise). iai-callgrind: deterministic instruction counts. Use black_box in benchmarks. Profile with flamegraph/cargo-instruments first. Perf crates: rayon, dashmap, smallvec, parking_lot.
