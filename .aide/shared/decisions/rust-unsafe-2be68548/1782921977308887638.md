# rust-unsafe

**Decision:** Avoid writing unsafe code; when necessary, minimize scope, document safety invariants, and test with Miri

## Rationale

Most unsafe functionality exists in the standard library or established crates. Unsafe blocks that lack safety documentation are time bombs. Miri detects undefined behavior that tests and sanitizers miss.

## Details

Prefer Rc/Arc/RefCell/Mutex/Pin over raw pointers. One op per unsafe block. Every block needs // SAFETY: comment. Wrap unsafe in safe-interface modules. Run cargo +nightly miri test. Enable clippy::undocumented_unsafe_blocks. Edition 2024 warns on unsafe_op_in_unsafe_fn by default.
